# V1.5 — Persistent Memory (Supabase) Plan

## Goal
Move from per-browser `localStorage` to **cloud-backed memory** so a user's Playbill, Playground saves, and history follow them across devices and persist over time ("gets smarter with every outing").

## Current state (what exists today)
- Everything is in **localStorage** (per-browser, per-origin):
  - `lastPlaybill` — the family profile/preferences
  - `softplay_playground` — saved cards (hearts/pins)
  - `softplay_vetoes_<city>` — flagged/vetoed suggestions
  - `softplay_notes_<id>` — itinerary notes
- ⚠️ **Bug found:** `lastPlaybill` is **read but never written** (play-plan/page.tsx reads it on mount; nothing ever calls `setItem('lastPlaybill', …)`). So "remember your Playbill" does nothing today. The Playground play-by-play path therefore always falls back to defaults (2 adults, empty prefs).

## Step 0 — Quick win (no Supabase, do first)
- **Persist the Playbill to localStorage** (write `lastPlaybill` whenever it's completed/updated). This makes "remember" work in-browser immediately, so the feel can be tested before the cloud work.
- This is the same data shape we'll later sync to Supabase, so it's not throwaway.

## Step 1 — Supabase project + connection
- Create a Supabase project (free tier).
- Add env vars in **Vercel → Settings → Environment Variables**: `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`.
- Domain/deploy unaffected — `mysoftplay.app` keeps working, deploys still via `npx vercel --prod --yes`.

## Step 2 — Auth (DECISION for Emily)
How do users identify themselves so memory is theirs?
- **Recommended for beta: email magic link** (passwordless) — tester enters email, clicks a link, done. Enables cross-device sync, no passwords.
- Alternative: anonymous device id (no login) — simplest, but no cross-device and easy to lose.
- Decision needed before building.

## Step 3 — Database schema (draft)
- `profiles` (1 row per user) — id, email, home_city, created_at
- `playbills` (1 per user) — user_id, the Playbill JSON (crew, fun/not-fun chips, food likes/avoids, notes)
- `saves` (Playground items) — user_id, title, emoji, pitch, city, type (heart|pin), created_at
- (later) `history` — chosen/flagged options, to power "gets smarter"
- Row-Level Security so each user only sees their own rows.

## Step 4 — Migrate reads/writes
- Replace localStorage reads/writes with Supabase calls; keep localStorage as an offline cache/fallback.
- **On first login, import existing localStorage data** into Supabase (so current testers don't lose their saves).

## Step 5 — Free Play personalization (DECISION for Emily)
- Today Free Play ignores the Playbill (`playbill: null`).
- Option A: feed the saved Playbill (crew, food avoids, dislikes) into Free Play so suggestions are personalized.
- Option B: keep Free Play quick/anonymous (no profile).
- Possible middle ground: a toggle ("use my preferences").

## Step 6 — "Gets smarter" (later, optional)
- Log chosen/flagged/saved outcomes and feed a short summary into future prompts ("you've loved markets and walks, avoided museums").

## Step 7 — Move testers over
- Once memory is live, migrate the 2 testers from `softplay-five.vercel.app` to `mysoftplay.app` (their localStorage saves are per-origin; the first-login import handles the carry-over, or they re-save).

## Open decisions to resolve at the start of V1.5

2. Free Play personalization (A / B / toggle).
3. How much "gets smarter" to attempt in V1.5 vs later.
